# Contributing to Boxwood

Thanks for your interest in contributing to Boxwood.

## Development setup

1. Clone the repository.
2. Install dependencies with `npm install`.
3. Build the package with `npm run build`.
4. Run the test suite if available.

## Project structure

- `resolve.ts` contains the main layout resolution logic.
- `split.ts` handles column, row, and grid splitting.
- `collide.ts` handles overlap resolution.
- `measure.ts` provides text measurement helpers.
- `examples/` contains demo scripts and HTML examples.

## Making changes

- Keep changes focused and well documented.
- Prefer small, composable updates.
- Update the README if the public API changes.
- Add or update tests when behavior changes.

## Pull requests

- Open a pull request with a clear summary of the change.
- Include context for why the change is useful.
- Mention any relevant examples or screenshots.

## Code of conduct

Be respectful and constructive in discussions and reviews.
